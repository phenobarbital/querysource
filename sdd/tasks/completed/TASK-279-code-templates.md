## Template Content

### `parrot/templates/agent.py.tpl`
```
"""$agent_name agent — generated by parrot setup."""
import logging
from parrot.bots import Agent


logger = logging.getLogger(__name__)


class $class_name(Agent):
    """$agent_name.

    Generated by parrot setup. Customize this class to add tools,
    system prompts, and other configuration.
    """

    agent_id: str = "$agent_id"
    agent_name: str = "$agent_name"

    def __init__(self, **kwargs) -> None:
        """Initialize $agent_name with the configured LLM."""
        super().__init__(
            name="$agent_name",
            agent_id="$agent_id",
            llm="$llm_string",
            **kwargs,
        )
```

### `parrot/templates/app.py.tpl`
```
"""Application entry point — generated by parrot setup."""
from parrot.manager import BotManager
from parrot.conf import STATIC_DIR
from parrot.handlers import AppHandler
from $agent_module import $class_name


class Main(AppHandler):
    """Main application handler.

    Add additional bot registrations and routes in configure().
    """

    app_name: str = "Parrot"
    enable_static: bool = True
    staticdir: str = STATIC_DIR

    def configure(self) -> None:
        """Configure bots and API routes."""
        self.bot_manager = BotManager()
        self.bot_manager.register($class_name())
        self.bot_manager.setup(self.app)
```

### `parrot/templates/run.py.tpl`
```
#!/usr/bin/env python3
"""Run script — generated by parrot setup."""
from navigator import Application
from app import Main

app = Application(Main, enable_jinja2=True)

if __name__ == "__main__":
    app.run()
```

## Implementation Notes

### `string.Template` escaping rules
- `$variable` → substituted with context value
- `$$` → renders as a single literal `$` in the output
- Templates must NOT contain bare `$` that aren't variable references — they will
  cause `KeyError` with `substitute()` (use `safe_substitute()` to be safe, which
  leaves unrecognized `$var` patterns untouched)

### Current `app.py` / `run.py` in the repo
The project root already contains `app.py` and `run.py`. The templates produce a
**minimal standalone version** suitable for a new project started from scratch.
The `bootstrap_app()` function (TASK-278) will skip writing if those files exist
and `--force` is not set.

### Key Constraints
- Templates are plain text files — no Python syntax errors inside them (they are
  not executed, only rendered).
- `agent.py.tpl` must produce valid importable Python after substitution.
- `run.py.tpl` must have the correct shebang line.
- No Jinja2 syntax — only `string.Template` `$variable` substitutions.
